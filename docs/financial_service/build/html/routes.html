Routes¶

routes.add_account()[source]¶

Creates a new account in the DynamoDB table using an API endpoint. Extracts ‘shop_id’ from the POST request JSON body.

Returns:

JSON response containing the creation status and the new account ID if successful, or an error message if not.

Raises:

ClientError – If there’s a DynamoDB client error during account creation.

routes.create_invoice(order_id)[source]¶

Generates an invoice for a completed order by interacting with an external order management service.

Parameters:

order_id – The unique identifier for the order for which the invoice is created.

Returns:

True if the invoice creation is successful, False otherwise.

routes.create_payment()[source]¶

Creates a Stripe PaymentIntent based on the total price and order ID received via POST request.

Returns:

JSON response with the clientSecret of the PaymentIntent for client-side use, or an error message.

Raises:

Exception – If there is an error during PaymentIntent creation.

routes.get_account(shop_id)[source]¶

Retrieves an account’s details by shop_id from the DynamoDB table. Logs the account data and handles retrieval errors.

Parameters:

shop_id – The unique identifier for the shop.

Returns:

JSON response containing the account details if found, or an error message if not.

Raises:

ClientError – If there’s a DynamoDB client error during account retrieval.

routes.get_order(order_id)[source]¶

Retrieves an order by its ID from an external orders service.

Parameters:

order_id – The unique identifier for the order.

Returns:

The order details if found, None otherwise.

routes.get_shop_from_product(product_id)[source]¶

Retrieves the shop owner of a product by querying an external inventory management service.

Parameters:

product_id – The unique identifier for the product.

Returns:

The shop owner if the product is found, None otherwise.

routes.get_user_mail(user_id)[source]¶

Fetches a user’s email by user_id from an external user service.

Parameters:

user_id – The unique identifier for the user.

Returns:

The user’s email if found, None otherwise.

routes.handle_payment_intent_succeeded(order_data, order_id)[source]¶

Processes a successful payment by allocating the total payment across sub-orders based on shop subtotals, and updates each shop’s balance. This function is invoked as part of the webhook handler for successful payments.

Parameters:

  • order_data – The data pertaining to the order including items and quantities.

  • order_id – The unique identifier for the order.

Returns:

JSON response indicating the success of the funds transfer across sub-orders.

routes.send_order_confirmation(email, order_id)[source]¶

Sends an order confirmation email to the user using the Flask-Mail extension. This function is called after a successful order payment is processed.

Parameters:

  • email – The recipient’s email address.

  • order_id – The unique identifier for the order.

Returns:

True if the email is sent successfully.

A simple test route that confirms the server is operational.

A JSON response containing a status indicator and a test message, along with HTTP status code 200.

Updates the payment status of an order to ‘paid’ by contacting an external order management service.

order_id – The unique identifier for the order whose status is to be updated.

Returns:

True if the update is successful, False otherwise.

routes.update_suborder_payed(order_id, product_owner_id)[source]¶

Updates the payment status of a suborder to ‘paid’. This function is called as part of the payment processing workflow.

  • order_id – The unique identifier for the order.

  • product_owner_id – The unique identifier for the product owner or shop.

routes.webhook()[source]¶

Handles incoming webhook events from Stripe, verifies the signature, processes the payment event, and updates order status if payment succeeds. Logs errors and events for diagnostics.

Returns:

JSON response indicating success or failure based on the event handling.

Raises:

  • stripe.error.SignatureVerificationError – If the signature verification of the webhook fails.

  • json.decoder.JSONDecodeError – If there is an error parsing the JSON payload from Stripe.
